OrderOfTheLionGM 1.5.7
=========================

CRAFTING ICONS AND RELIABILITY
- Added persistent bounded caches for recipe and reagent textures.
- Local scans prefer direct Trade Skill/Craft window texture APIs.
- RC3 preserves recipe icons, reagent icons, links, quality and materials.
- Added manifest sync comparing owner/profession/hash/count and completeness.
- Requests only missing, changed or more complete profession snapshots.
- Allows targeted relay of a complete cached profession when its owner is offline.
- Existing complete data remains visible until a replacement snapshot is whole.
- Same-hash incomplete icon/material data can upgrade from a better source.

PROFESSIONS INTERACTION
- Left-click a crafter to whisper that exact character.
- Right-click a crafter for Whisper, Invite to Group and View in Roster.
- Invite is disabled while the crafter is offline.
- Removed duplicated generic crafter-whisper action.

RAID PLANNER
- Fully hides legacy raid publication controls and text.
- Added live weekday/full-date/[HH:MM ST] preview.
- Added distinct Main Raid icon, styling and pinned planner position.
- Home lists up to three active raids chronologically and marks Main Raids.
- Cancelled events remain visibly marked in Upcoming until scheduled time.
- Seen remains available to all; Ready requires approved raid rank.
- Main/cancellation metadata travels in a separate bounded packet.

ANNOUNCEMENTS / ACTIVITY / CHAT
- New Announcement always opens with an empty form.
- Added Important and Publications activity filters.
- Added colored RAID/POST/GROUP/CRAFT/REPLY/REACT activity labels.
- Guild Chat player menu survives incoming messages.
- Repeat right-click, outside click, Escape, action or tab switch closes it.

OPTIMIZATION
- No new permanent polling loop.
- Manifest packets and RC3 chunks remain under the Vanilla 250-byte limit.
- Icon-cache cleanup runs infrequently and trims only fallback indexes.
- SavedVariables schema updated to 12.

OrderOfTheLionGM 1.5.6

RAID PLANNER
- Replaced the permanent Raid Alerts editor with an Upcoming / Cancelled / Past schedule and a separate detail panel.
- Create Raid always generates a new event ID; Edit updates only the selected event.
- Added exact Server Time, gathering time, one-time/weekly recurrence and per-event reminder lead time.
- Added Duplicate, Post to /g, Remind, Cancel and Permanent Delete under a compact More menu.
- Added schedule paging and mouse-wheel navigation for larger raid lists.
- Added 30-day raid deletion tombstones and fixed the legacy active-raid pointer that could restore a cancelled/deleted event after synchronization.

PROFESSIONS AND CRAFTING
- Rebuilt mismatched legacy profession keys into stable internal keys so recipes appear in their actual profession, not only All Professions.
- Simplified the page with a compact search clear control, a collapsible Filters panel, visible common categories and a More category menu.
- Kept Request This Craft as the primary action and grouped Whisper, Link Item, Link Recipe and Crafting Activity under More.
- Added clear unavailable states for missing item/recipe links or a missing selected crafter.
- Preserved RC3 metadata, quality colours, reagent recovery, fallback icons and immediate event-driven refresh from 1.5.5.

SHARED ACTIVITY
- Added privacy-safe sharing of online-count observations between addon users.
- Records at most one local sample per 15-minute interval and never sends roster names, classes or zones.
- Added anonymous source IDs, interval de-duplication, average/peak calculations, coverage percentage and source count.
- Replaced a potentially huge interval history response with at most one compact hourly packet per day (14 packets per responding client).
- Added a manual Shared Activity sync button and automatic initial recovery sync.

VISIBLE OPERATION STATES
- Update Roster, Crafting Sync, PvE Sync and Shared Activity Sync now disable while working and reject repeated clicks.
- Added visible Updating/Syncing, Complete and Retry states with clearer status text.
- Added a roster timeout so an unanswered server request does not appear active forever.

GUILD CHAT
- The right-click player menu now survives incoming Guild Chat refreshes and remains usable during busy conversation.
- The menu still closes normally after an action, manual dismissal, Escape or leaving the chat view.

NETWORK AND PERFORMANCE
- Kept one bounded one-second heartbeat instead of adding new continuous OnUpdate scanners.
- Shared activity sends one live packet per changed interval and compact history only for initial/manual recovery.
- Filters remain local and never trigger network traffic.
- Open pages refresh only after their underlying logical object changes or a complete snapshot finishes.
- Preserved bounded PvE/Community queues, item-cache retries, histories and cleanup windows.

DATA AND DIAGNOSTICS
- Advanced SavedVariables schema to 11 with defensive migration of existing 1.5.x data.
- Added /otltest markers for Quality 1.5.6, Raid Planner and Shared Activity.
- Updated README, release notes, acceptance checklist and validation report.

VERSION 1.5.3

ANNOUNCEMENT COMPOSER
- Fixed the modal frame hierarchy that could place Title and Message fields under the dark overlay.
- Rebuilt the composer with explicit FULLSCREEN_DIALOG strata and child frame levels.
- Added visible opaque input backgrounds and focus borders.
- Added title/body validation and disabled Publish until both required fields are present.
- Added local draft retention for unfinished new announcements.
- Prevented editing or cancelling an existing announcement from overwriting the separate new-post draft.

LEADERSHIP ONLINE
- Added distinct role icons for Guild Leader, Raid Leader, Officer and Helper.
- Added role/class hover information and preserved click-to-whisper.
- Sorted online leadership by role priority, then rank index and name.

FULL ACTIVITY WINDOWS
- Added a full Guild Activity modal opened from Home -> Recent Useful Activity -> View All.
- Added filters for Groups, Crafting, Replies and Reactions.
- Added a full Crafting Activity modal opened from Professions -> Recent Crafting.
- Added filters for Recipes, Requests, Replies and Reactions.
- Added paging, bounded 50-entry views, timestamps and click-through to the related main page.
- Added reactions on the player's own records to Recent Useful Activity.

PROFESSION NAVIGATION AND FILTERING
- Added dynamic category tabs that change with the selected profession.
- Added slot-based Enchanting categories: Weapon, Chest, Bracers, Gloves, Boots, Cloak, Shield, Legs and Other.
- Added practical category sets for Alchemy, Blacksmithing, Tailoring, Leatherworking, Engineering, Cooking, Jewelcrafting and Mining.
- Added required-level filters: Any, 1-20, 21-40, 41-59, 60 and Unknown.
- Added rarity filters: Any, Common, Uncommon, Rare, Epic and Unknown.
- Added sorting by online availability, name, required level, rarity, recent scan and crafter count.
- Added Online: All / Only filtering.
- Reduced visible recipe rows to eight to reserve a stable filter area without overlap.
- Preserved recipes with unavailable item-cache data under Unknown.

CRAFTING METADATA
- Extended local profession scans with item level, required level, item type, subtype and equipment slot when cached.
- Corrected item texture extraction to use the Vanilla GetItemInfo texture return.
- Restored cached remote metadata when an item ID is available locally.

WORLD CHANNEL
- Added automatic World-channel detection by channel name and the current joined-channel list.
- Recruitment now displays AUTO /n with the real client-assigned channel number.
- World send buttons use the detected number rather than assuming /5 or /6.
- Kept a clearly labelled manual fallback when automatic detection is unavailable.
- Refreshes detection on login, entering the world, channel notices and Recruitment refresh.

DIAGNOSTICS AND DATA
- Advanced SavedVariables schema from 7 to 8.
- Added defaults for announcement draft and profession-filter preferences.
- Added diagnostics for World channel, useful activity count and active profession filters.
- Corrected the UI loading marker to report ui153Loaded.
- Preserved existing roster, history, PvE, crafting, notification and announcement data during migration.

PERFORMANCE / COMPATIBILITY
- No new unbounded lists.
- Activity readers cap results at 50 and display ten rows per page.
- Filters operate on the selected profession result set and do not introduce a new per-frame scan.
- Existing search debounce and coarse heartbeat remain the only timed UI/background work.
- No frames are created inside Refresh functions.
- Interface target remains 11200.
